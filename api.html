03 · IRIS-Edge 2.0

API Documentation

IRIS-Edge 2.0 은 하드웨어 제어와 워크플로우 실행 로직을 REST · WebSocket · gRPC API 로 노출합니다. 어떤 프론트엔드(웹·모바일·헤드리스)에서도 동일한 산업 자동화 로직을 구동할 수 있습니다.

gRPC API

REST 와 동일한 기능을 고성능 스트리밍 RPC 로 제공합니다 (포트 50051).

서비스주요 스트리밍 RPC
CameraServiceStream — 서버 스트리밍 JPEG
RobotServiceStreamPose — 실시간 포즈 피드 · GetAnalogInput — 아날로그 입력 유너리 (⚠️ UR 전용)
AIService요청/응답 추론
PLCServiceWaitSignal — 블로킹 유너리
DeviceServiceModbusWaitSignal — 블로킹 유너리
NodeServiceSubscribeEvents, Pause, Resume, GetEngineStatus, GetResults, GetSequence
RobotService.GetAnalogInput — 로봇 아날로그 입력값 읽기. 요청은 로봇 key + channel(UR: 0 또는 1), 응답은 status + double value 입니다. ⚠️ UR 로봇 전용이며, 그 외 로봇 타입에서 호출하면 에러가 반환됩니다.

노드 프로젝트 JSON

노드 그래프는 nodes 와 connections 두 배열로 표현됩니다.

{
  "nodes": [
    {"uuid": "n1", "type": "Start", "settings": {}, "x": 0,   "y": 0},
    {"uuid": "n2", "type": "Delay", "settings": {"delay_time": 1000}, "x": 200, "y": 0},
    {"uuid": "n3", "type": "End",   "settings": {}, "x": 400, "y": 0}
  ],
  "connections": [
    {"start_id": "n1", "start_pin": "Ex Out",   "end_id": "n2", "end_pin": "Ex In"},
    {"start_id": "n2", "start_pin": "finished", "end_id": "n3", "end_pin": "Ex In"}
  ]
}
  • 노드 키 — uuid(아이디), type(노드 타입), settings(설정 dict). title·x·y 는 UI 표시용(실행 무영향).
  • 연결 키 — start_id/start_pin (출력) → end_id/end_pin (입력).
  • 실행 흐름 핀 — 입력 Ex In, 출력 finished/failed. Start 노드의 출력은 Ex Out.
IRIS-Edge(데스크탑)에서 만든 레거시 프로젝트 JSON 도 그대로 로드할 수 있습니다. 옛 타입 이름·설정 키는 현재 규격으로 자동 정규화됩니다.

WebSocket 이벤트 스트림

ws://<host>:8000/node/events 에 연결하면 실행 중 이벤트를 실시간 수신합니다.

{
  "type": "node_finished",
  "node_id": "n2",
  "node_type": "Delay",
  "node_title": "지연 노드",
  "message": "",
  "timestamp_unix_ms": 1715000000000,
  "run_time_ms": 1002.3,
  "loop_iteration": 3,
  "data": {"status": "finished"}
}

이벤트 타입: node_started, node_finished, node_error, node_log, execution_started, execution_stopped, execution_loop_iteration, execution_paused, execution_resumed
