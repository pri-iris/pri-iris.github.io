Socket Message Protocol

IRIS-Edge 는 로봇·디바이스와 TCP 소켓으로 통신합니다. 본 문서는 소켓으로 주고받는 메시지의 프레이밍(framing) 규격을 정의합니다. 외부 로봇·컨트롤러를 연동할 때, 장비 측이 아래 규격에 맞춰 메시지를 구성·해석하면 됩니다.

개요

통신은 양방향 TCP 스트림이며, 메시지 경계(framing)는 다음 요소의 조합으로 표현됩니다.

요소값설명
STX0x02메시지 시작 바이트 (Start of Text)
ETX0x03메시지 종료 바이트 (End of Text)
CR0x0D (\r)Carriage Return
LF0x0A (\n)Line Feed — CRLF 모드의 구분자
Header사용자 지정메시지 앞에 붙는 문자열/바이트 (선택)
Footer사용자 지정메시지 뒤에 붙는 문자열/바이트 (선택)

인코딩은 기본적으로 UTF-8 이며, 바이너리(bytes) 페이로드도 전송할 수 있습니다.

송신 (Send)

송신 메시지는 다음 순서로 조립됩니다. STX·ETX 와 CRLF 는 옵션이며, Header/Footer 는 지정 시에만 부착됩니다.

Header STX 0x02 PAYLOAD CRLF \r\n ETX 0x03 Footer

조립 규칙 (안쪽 → 바깥쪽):

  1. PAYLOAD — 전송할 본문 (문자열 또는 bytes).
  2. CRLF 옵션이 켜져 있으면 PAYLOAD 뒤에 \r\n 을 덧붙입니다. (기본 ON)
  3. STX/ETX 옵션이 켜져 있으면 그 결과를 0x02 … 0x03 으로 감쌉니다. (기본 OFF)
  4. Header 가 지정되면 맨 앞에 붙입니다.
  5. Footer 가 지정되면 맨 뒤에 붙입니다.
기본 설정(CRLF ON · STX/ETX OFF · Header/Footer 없음)에서 문자열 RESULT_OK 를 보내면 실제 전송 바이트는 RESULT_OK\r\n 입니다.
송신 옵션기본값동작
노드 입력 사용OFF이전 단계 결과값을 페이로드로 사용
CRLF 부착ON페이로드 끝에 \r\n 추가
STX + ETX 부착OFF0x02 … 0x03 으로 감싸기
Header없음메시지 앞 문자열/HEX
Footer없음메시지 뒤 문자열/HEX

수신 (Receive)

수신은 들어오는 바이트 스트림을 버퍼에 누적한 뒤, 선택한 파서 모드에 따라 메시지 경계를 잘라 한 건씩 꺼냅니다. 디코딩된 텍스트는 UTF-8 로 해석되고 앞뒤 공백이 제거됩니다.

수신 페이로드가 유효한 JSON 이면 자동으로 보기 좋게 재포맷되어 전달됩니다(레거시 호환). 바이너리(Bytes) 모드의 결과는 원본 바이트로 유지됩니다.

수신 파서 모드

장비가 보내는 메시지 형식에 맞춰 다음 네 가지 모드 중 하나를 선택합니다.

모드경계 규칙설명
None 버퍼 전체 수신 시점의 버퍼를 한 메시지로 그대로 반환 (구분자 없음). 기본값
CRLF \n 단위 분할 \n 으로 라인을 나누고 끝의 \r 을 제거
STX + ETX 0x02 … 0x03 STX 와 ETX 사이의 내용을 추출 (바이너리 프레이밍)
Bytes 고정 길이 지정한 Byte Length 만큼의 고정 길이 청크로 분할 (raw bytes)
Bytes 모드는 정확한 바이트 수가 모일 때까지 대기한 뒤 그 길이만큼 잘라냅니다. 수신할 메시지의 길이가 고정된 바이너리 프로토콜에 사용하며, Byte Length 를 반드시 지정해야 합니다.

Header / Footer 의 HEX 표기

Header 와 Footer 는 일반 문자열뿐 아니라 HEX 바이트로도 지정할 수 있습니다. 값이 \x 로 시작하면 16진수 바이트로 해석됩니다.

입력 값해석전송 바이트
ABC문자열0x41 0x42 0x43
\x02HEX 1바이트0x02
\x0D0AHEX 2바이트0x0D 0x0A

IRIS Image 전송 프로토콜

스캔 이미지를 전송하기 위한 바이너리 가변 길이 프로토콜입니다. 앞서 설명한 텍스트 기반 프레이밍과 달리, 메타데이터와 JPEG 이미지 데이터를 하나의 패킷으로 묶어 전달합니다. 전용 수신 모드(IMAGE_TRANSFER)로 동작하며 기본 포트는 8080 입니다.

모든 정수 필드는 Little-Endian 이며, 문자열 필드는 [4바이트 길이][UTF-8 본문] 형태의 length-prefixed 구조입니다. 수신측은 완전한 패킷이 버퍼에 모일 때까지 대기한 뒤 한 패킷씩 처리합니다.

패킷 구조

#필드크기타입설명
1MessageType Length4 Bint32 LE메시지 타입 문자열 길이 (N)
2MessageTypeN ByteUTF-8메시지 타입 문자열 (N)
3CameraName Length4 Bint32 LE카메라 이름 길이 (N)
4CameraNameN ByteUTF-8촬영 카메라 이름 (N)
5ScanID8 Bint64 LE스캔 식별자
6ScanResult4 Bint32 LE스캔 결과 값
7ImagePath Length4 Bint32 LE이미지 경로 길이 (N)
8ImagePathN ByteUTF-8이미지 파일 경로 (N)
9ImageData Length4 Bint32 LE이미지 데이터 길이 (N)
10ImageDataN ByteJPG binaryJPEG 이미지 바이트 (N)

바이트 레이아웃

len + MessageType len + CameraName ScanID (8B) ScanResult (4B) len + ImagePath len + ImageData (JPG)

수신 결과 필드

한 패킷을 수신하면 다음 필드로 분해되어 전달됩니다.

필드설명
message_type메시지 타입
camera_name카메라 이름
scan_id스캔 식별자 (int64)
scan_result스캔 결과 (int32)
image_path이미지 파일 경로
image_dataJPEG 이미지 바이트
image_size이미지 데이터 길이 (바이트)
각 길이 필드(문자열 길이·ImageData Length)는 안전을 위해 최대 20 MB 로 제한됩니다. 범위를 벗어난 길이가 수신되면 비정상 패킷으로 간주하여 버퍼를 초기화합니다.

수신 큐 초기화

소켓 수신 데이터는 내부 큐에 누적됩니다. 시퀀스 시작 전 묵은 데이터로 인한 오동작을 막기 위해 수신 큐를 비우는(Clear) 동작을 제공합니다. (워크플로우의 Socket 큐 초기화 노드에 해당)

예시

예시 1 — 기본 텍스트 (CRLF)

판정 결과 OK 를 기본 설정으로 송신:

송신 옵션 : CRLF=ON, STX/ETX=OFF
페이로드  : "OK"
전송 바이트 : 4F 4B 0D 0A          →  O K \r \n

예시 2 — STX/ETX 프레이밍

좌표 문자열을 STX/ETX 로 감싸 송신(CRLF 포함):

송신 옵션 : CRLF=ON, STX/ETX=ON
페이로드  : "100,200,30"
전송 바이트 : 02 31 30 30 2C 32 30 30 2C 33 30 0D 0A 03
            STX  1  0  0  ,  2  0  0  ,  3  0  \r \n ETX

예시 3 — Header + Footer (HEX)

Header  : \x02
Footer  : \x03
CRLF    : OFF
페이로드 : "MOVE"
전송 바이트 : 02 4D 4F 56 45 03     →  STX M O V E ETX

예시 4 — 고정 길이 수신 (Bytes)

수신 모드   : Bytes
Byte Length : 16
동작        : 버퍼에 16바이트가 모이면 정확히 16바이트를 잘라 raw bytes 로 반환
송신과 수신은 독립적으로 설정합니다. 예를 들어 장비가 STX/ETX 로 보내고 CRLF 로 응답을 기대한다면, 수신은 STX + ETX 모드, 송신은 CRLF 옵션을 켜서 맞춥니다.
