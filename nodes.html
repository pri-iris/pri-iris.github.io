사용 가능한 노드

IRIS-Edge 와 IRIS-Edge 2.0 의 워크플로우는 노드(Node)를 연결해 구성합니다. 각 노드는 실행 흐름 핀(Ex In → finished / failed)으로 이어지고, 데이터 핀으로 값을 다음 노드에 전달합니다. 아래는 카테고리별 노드 목록입니다.

실행 흐름은 Ex In 으로 들어와 처리 후 성공 시 finished, 실패 시 failed 핀으로 분기합니다. 데이터 핀은 노드 간 값(이미지·좌표·판정 등)을 전달합니다.

IRIS-Edge 2.0

IRIS-Edge 2.0 노드

IRIS-Edge 2.0 은 동일한 노드 엔진을 API 로 구동하며, 내장 노드 외에 플러그인 노드를 파일로 추가할 수 있습니다. 실시간 전체 팔레트는 GET /node/catalog 로 조회합니다. 카테고리별 내장 노드는 다음과 같습니다.

카테고리노드 타입
제어 흐름 Start · End · Init · Delay · Compare · Judgment · Condition · Switch · Branch · 내부 신호 대기 · 내부 신호 쓰기 · 내부 신호 읽기
카메라 CameraGrab · CameraSetting
로봇 RobotMove · RobotMotion · RobotPose · RobotDigitalInOut · RobotAnalogIn · SocketInit · SocketSend · SocketReceive
AI AIInference · AIResultSelect · AIROIFilter · ImageProcessingResult · ImageProcessingGetDepth · ImageProcessingCoordTransform2D · ImageProcessingReadQR
PLC PLCRead · PLCWrite · PLCWaitSignal
디바이스 DeviceModbusRead · DeviceModbusWrite · DeviceModbusWaitSignal · DeviceSocketInit · DeviceSocketSend · DeviceSocketReceive · DeviceHttpRequest · LightControl
ROS2 ROSTopicPublish · ROSTopicSubscribe · ROSServiceClient · ROSActionClient
사용자 정의 UserNode · UserInputNode · PluginNode
동적 핀 — Switch·Branch·PluginNode 는 설정에 따라 출력 핀이 달라집니다.
추론 후처리 — AIResultSelect(클래스·신뢰도 선택)·AIROIFilter(ROI 내부만 통과)는 YOLO·SAM2·FastSAM·이상감지·깊이 등 모델 종류와 무관하게 동작합니다.
로봇 경로 이동 — RobotMotion(로봇 경로 이동)은 포즈 목록을 순회하며 자세마다 선형 이동(movel)합니다.
HTTP 요청 — DeviceHttpRequest(HTTP 요청)는 http 디바이스로 요청을 보내고 상태코드·응답 본문을 반환합니다.
⚠️ RobotAnalogIn(로봇 아날로그 입력)은 UR 로봇 전용입니다 (UR / UR RTDE). 그 외 로봇 타입은 지원하지 않습니다.
